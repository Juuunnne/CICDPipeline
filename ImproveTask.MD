## 3. Detailed Implementation Notes

### T1 â€“ Enable write-permissions
1. Add at the **top-level** of `.github/workflows/ci-cd.yml`:
   ```yaml
   permissions:
     contents: write  # allows pushing commits/tags & creating releases
   ```
2. No additional secrets required â€“ `GITHUB_TOKEN` inherits write access when this permission is set.

### T2 â€“ Push commit & tag
1. Insert a step **immediately after** `npm run release`:
   ```yaml
   - name: Push version commit & tag
     run: |
       git push --follow-tags origin master
   ```
2. Uses default `GITHUB_TOKEN` â€“ no PAT necessary.

### T4 â€“ Add `release-tag.yml`

```yaml

  push:
    tags:
      - "v*"

      - run: npm run build

        uses: ncipollo/release-action@v1
        with:
          artifacts: cicdpipeline.zip
```
